## О проекте

Данный проект представляет собой решение для автоматического обнаружения переломов костей на рентгеновских снимках с использованием компьютерного зрения. Система основана на архитектуре **YOLOv8** (You Only Look Once).

### Что входит в проект:

| Компонент | Описание |
|-----------|----------|
| **Обученная модель YOLOv8** | Модель для детекции переломов на рентгеновских снимках |
| **FastAPI бэкенд** | API для инференса модели |
| **Streamlit веб-интерфейс** | Базовый веб-интерфейс для загрузки и анализа изображений |
| **Docker** | Kонтейнеры для развертывания |


### Цель проекта

Научиться писать веб-приложения под готовую обученную CV-модель


## Возможности

### Основные функции

- Быстрое обнаружение переломов на загруженных рентгеновских снимках
- API с автоматической документацией
- Веб-интерфейс для загрузки и визуализации результатов
- Оценка уверенности для каждого обнаруженного перелома (confidence score)
- Визуализация bounding boxes
- Контейнеризация через Docker для production-развертывания
- Поддержка различных форматов изображений (JPG, JPEG, PNG)

### Технические особенности

- Асинхронная обработка запросов
- CORS поддержка для интеграции с другими приложениями
- Подробное логирование всех операций
- Health check эндпоинт для мониторинга

## Метрики модели

Модель обучена на датасете рентгеновских снимков с переломами. Результаты валидации на тестовой выборке:

| Метрика | Значение | Описание |
|---------|----------|----------|
| **mAP50** | 0.0080 | Средняя точность при IoU=0.5 |
| **mAP50-95** | 0.0023 | Средняя точность при разных порогах IoU |
| **Precision** | 0.0047 | Точность предсказанных переломов |
| **Recall** | 0.6471 | Полнота — модель находит правильно из всех переломов |

## Установка

### Системные требования

- Python: версия 3.8 или выше
- GPU:опционально (для ускорения инференса)

### Вариант 1: Локальная установка

#### Шаг 1: Клонирование репозитория

```bash
git clone https://github.com/ukiGlenny/bone_fracture_detection.git
cd bone_fracture_detection
```

#### Шаг 2: Создание виртуального окружения

```bash
#На macOS/Linux
python3.8 -m venv venv
source venv/bin/activate

#На Windows
python -m venv venv
venv\Scripts\activate
```

#### Шаг 3: Установка зависимостей бэкенда
```bash
cd backend
pip install -r requirements.txt
```

#### Шаг 4: Установка зависимостей фронтенда
```bash
cd ../frontend
pip install -r requirements.txt
```

#### Шаг 5: Запустить backend
```bash
python ../backend/app.py
```    
 
#### Шаг 6: Во втором терминале запустить frontend
```bash
streamlit run frontend/streamlit_app.py
```
